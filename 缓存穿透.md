# 链路 1：限流热路径（AOP 切面）— 没有穿透问题

```
每个 HTTP 请求
       ↓
RateLimitAspect.around()
       ↓
resolveEffectiveConfig(rawKey, rateLimit)
       ↓
rateLimitConfigService.getActiveConfig(rawKey)   ← 第 54-56 行
       ↓
cache.get(limitKey)   ← 只查 ConcurrentHashMap，不查 DB
       ↓
null → 回退到注解配置（不会穿透到 DB）
```

`getActiveConfig()` 只读本地内存缓存，cache miss 时直接返回 `null`，由切面回退到注解默认值。无论传什么 key，永远不会打到 DB。这条热路径在设计上是**天然免疫缓存穿透**的。
